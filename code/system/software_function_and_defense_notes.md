# FusionTrack 软件功能内容与答辩准备说明

本文档用于毕业设计展示、软件验收材料整理和答辩前准备。当前系统定位为“多模态目标轨迹融合与异常检测结果展示系统”：它不是单一算法脚本，而是把数据集、异常标签协议、个体异常检测、群体异常检测、配准诊断、实验结果汇总和可视化展示整合到同一个静态网页交付物中。

## 1. 软件基本定位

FusionTrack 系统面向 VT-Tiny-MOT 多模态目标跟踪数据，围绕 RGB / Thermal 轨迹融合、异常检测实验结果分析和可解释可视化展示进行设计。当前软件更偏向论文展示和实验归档层：后端负责汇总实验产物、统一结构化结果、脱敏路径并生成静态 dashboard；前端负责展示指标、播放轨迹、解释异常证据、导出当前视图和下载完整报告包。

系统当前采用离线生成、静态发布的方式运行。也就是说，实验算法和结果汇总先在本地或服务器上跑完，然后由 `code/system/run_fusiontrack.py` 生成 `index.html` 与 `assets/`，最后可以部署到 GitHub Pages 作为公开静态网页。网页本身不依赖在线后端服务，便于答辩现场离线打开，也便于公开展示。

## 2. 软件主要功能

### 2.1 数据接入与数据审计

系统支持接入 VT-Tiny-MOT 数据集及其轨迹产物。后端会生成 `dataset_manifest_<split>.json`，记录数据集名称、数据根目录状态、图像/标注目录、annotation 文件摘要、split 信息和数据指纹。这样可以说明每次展示结果来自哪一批数据，避免答辩时只展示静态图但无法解释数据来源。

网页中的“数据流审计”面板会继续检查可视化所需数据是否完整，包括：

- 当前任务下可播放序列数量。
- 可视化轨迹数量。
- 轨迹点数量与帧范围。
- 分数覆盖情况。
- 标签覆盖情况。
- RGB / Thermal 模态覆盖率。
- 背景帧是否存在。
- 原始视频背景、点云配准背景或无背景轨迹模式的媒体类型。

这部分可以用于回答“系统是否真的接入了数据集”“为什么某些视频没有原始背景”“展示用的数据是否完整”等问题。

### 2.2 异常标签协议管理

VT-Tiny-MOT 原始数据没有直接可用于异常检测的异常标签，因此当前实验使用规则化 synthetic anomaly injection 构造异常标签。系统会把异常协议 manifest 接入 dashboard 的 provenance 区域，说明标签来源、异常比例、异常类型、正样本数量和标签总数。

当前支持的异常协议主要分为：

- 个体异常：路线偏移、速度突变、停滞或异常减速、轨迹跳变、形状扰动、RGB/Thermal 模态偏移等。
- 群体异常：离群、逆向运动、邻居替换、群体数量变化、离散程度变化、群体分裂或合并等。
- 配准诊断：作为几何对齐能力评估模块，展示旋转误差、平移误差、Chamfer 距离、耗时和成功率；它不是 synthetic anomaly label 任务。

答辩时需要明确说明：当前异常并非数据集原生标注，而是根据论文系统需要构造的规则化异常协议；后续如果有人工异常标注或真实异常数据，只需要把标签文件和 score row 按相同 key policy 接入即可。

### 2.3 实验结果汇总与方法管理

系统会读取最终实验结果目录下的 summary CSV / JSON、score JSONL 和方法注册表，生成统一的 leaderboard、case 分析和方法状态表。方法注册表 `method_registry.json` 是方法身份的权威来源，用于统一管理：

- 方法名称。
- 方法角色：主方法、组件、基线等。
- 方法来源：FusionTrack、经典算法、本地规则基线、外部实验产物等。
- 学习类型：学习类方法、非学习规则方法、经典机器学习方法等。
- 方法族：如 nearest-neighbor profile、LOF、linear prediction residual、registration baseline 等。
- 接入状态：已接入、占位、待增强、未注册等。

这种设计可以避免不同实验 CSV 中方法名称和分类不一致，保证页面展示、实验表格和 README 文档之间的一致性。

### 2.4 个体异常检测展示

个体异常检测展示围绕单条轨迹进行，前端可以按任务、方法和序列筛选结果。页面会显示当前方法的 AUROC、AUPRC、F1、Precision@K、Recall@K 等指标，并在可视化区域展示：

- 原始 RGB 背景。
- 当前轨迹路径。
- 异常热力区域。
- 轨迹分数和异常标签。
- 当前选中轨迹的解释证据。
- 轨迹长度、平均速度、位移等运动特征。
- route / speed / shape 等子模块证据。

当前轻量个体检测器会输出 `component_scores`、`frame_event_scores`、`event_segments` 和 `explanation_schema`，页面会进一步展示分数分解、事件时间线和当前时间窗口内的异常事件证据。后续如果替换为更强的学习模型，也应继续输出同样字段，从而不需要重写展示层。

### 2.5 群体异常检测展示

群体异常检测展示关注多目标之间的关系变化。页面除了轨迹和热力图外，还会绘制群体关系辅助图，包括当前帧群体中心、群体半径和目标到群体中心的连接线。对于群体事件，页面可以展示：

- 涉及轨迹数量。
- 事件持续帧数。
- 群体中心位置。
- 主导异常原因。
- 群体事件聚合结果。
- 高风险群体轨迹排序。

这部分用于说明系统不仅关注单个目标的运动异常，也关注多目标关系结构的异常变化。

### 2.6 配准诊断展示

Registration 模块用于展示几何配准能力，不使用 VT-Tiny-MOT 原始视频背景。切换到 Registration 任务后，页面显示点云配准动态画布和配准证据面板，包括：

- source 点云。
- reference 点云。
- aligned 估计对齐结果。
- 当前点云对编号。
- 配准方法。
- 旋转误差。
- 平移误差。
- Chamfer 距离。
- 运行耗时。
- 成功或失败状态。

如果当前 score row 暂时没有真实 `registration_points`，页面会使用轻量占位点云保证展示层不断裂。后续接入真实 MPS-GAF 或其他学习式配准模型输出后，可以用真实点云替换占位数据。

### 2.7 动态可视化与交互

网页默认采用四画面对比展示，包含：

- 原视频：只显示原始 RGB 背景帧。
- 热力图：在原始背景上叠加异常热力。
- 轨迹图：在原始背景上叠加目标轨迹。
- 热力 + 轨迹：同时叠加热力和轨迹，作为默认综合观察视图。

页面使用 Canvas 逐帧绘制，而不是直接播放 MP4。这样可以把轨迹、热力、群体关系和解释证据全部绑定到同一帧控制上。支持的交互包括：

- 播放 / 暂停。
- 帧滑块拖动。
- 播放速度调节。
- 热力透明度调节。
- 时间窗口调节。
- 事件阈值调节。
- 四画面对比和单画面模式切换。
- 单画面图层切换。
- 点击画布中的轨迹选择解释对象。
- 键盘快捷控制播放、逐帧、跳转和切换视图。
- 中文 / English 切换。
- 论文展示版 / 完整审计版切换。

### 2.8 解释与证据链

系统后端会统一构造 `explanation_schema`，前端直接渲染解释结构，避免每个页面模块自行猜测异常原因。解释链包括：

- 选中轨迹或点云对。
- 异常分数。
- 异常类型。
- 标签帧段或配准误差。
- 主导原因。
- 证据来源：event segment、frame event scores、score components 或 registration metrics。
- 分数分解：个体分数、群体分数、事件分数、融合分数。
- 当前时间窗口内的异常事件统计。
- 事件时间线：真实异常段、预测异常段和重叠区域。

这部分可以用于答辩时说明“系统不是只给一个异常分数，而是能解释为什么当前轨迹被判为异常”。

### 2.9 数据导出与报告包交付

审计版页面提供轻量导出功能：

- 导出当前视图 JSON。
- 导出当前任务排行榜 CSV。
- 导出当前序列 JSON。
- 导出当前播放画面 PNG。
- 下载完整报告包 ZIP。

当构建命令传入 `--export-package` 时，系统会生成完整 ZIP 报告包，并把该 ZIP 复制到 dashboard 的 `assets/` 目录。网页右上角会出现“下载完整报告包”按钮，GitHub Pages 发布时也会把 ZIP 一起发布到 `assets/<zip_name>`。报告包中包含 dashboard、脱敏后的 pipeline summary、pipeline manifest、dataset manifest、protocol manifest、suite/holdout/fused pipeline 等被引用的实验产物。

这部分适合答辩归档：老师或评阅人可以打开网页看结果，也可以下载 ZIP 离线检查结果来源。

## 3. 软件模块结构

当前系统层主要代码位于 `code/system`：

- `run_fusiontrack.py`：统一运行入口，负责读取配置、调用 pipeline、生成 dashboard 和导出包。
- `fusiontrack/pipeline.py`：系统流程编排，负责数据 manifest、最终结果读取、dashboard 构建、远程报告同步等。
- `fusiontrack/final_dashboard.py`：最终网页生成器，包含 HTML、CSS、JS、播放数据、前端交互和 provenance 渲染。
- `fusiontrack/final_results.py`：最终实验结果加载、方法排行榜、异常类型统计和典型案例提取。
- `fusiontrack/export_package.py`：完整报告包打包与路径脱敏。
- `fusiontrack/dataset_manifest.py`：数据集 manifest 与数据指纹生成。
- `fusiontrack/explanation_schema.py`：统一解释结构生成。
- `fusiontrack/event_segments.py`：逐帧事件证据标准化和事件段合并。
- `fusiontrack/method_registry.py`：方法注册表读取与校验。
- `tools/publish_dashboard_pages.py`：将 dashboard 发布到 GitHub Pages 工作树。
- `tools/build_dashboard_release.py`：一键构建 dashboard、生成导出包、发布 Pages、写 release manifest。

## 4. 推荐答辩演示流程

1. 打开 GitHub Pages 或本地 `index.html`，先说明系统整体定位：多模态轨迹融合与异常检测结果展示系统。
2. 在顶部切换展示模式，先使用“论文展示版”，展示系统流程、结果快照和证据快照。
3. 展示 Individual 任务，播放一个真实 VT-Tiny-MOT 序列，说明四画面对比、轨迹热力和异常解释。
4. 切换 Group 任务，展示群体关系、群体中心、群体半径和群体事件。
5. 切换 Registration 任务，说明该模块是几何配准诊断，不是原始视频异常检测任务。
6. 打开完整审计版，展示排行榜、异常类型分析、典型案例、算法接入状态和数据流审计。
7. 展示中文 / English 切换，说明页面可用于公开展示。
8. 点击导出当前画面 PNG 或下载完整报告包，说明系统可以归档和离线复查。

## 5. 答辩老师可能关注的问题

### 5.1 当前异常是不是人工设计的？

是。VT-Tiny-MOT 当前没有异常标签，系统使用规则化 synthetic anomaly injection 构造个体和群体异常标签。该设计用于搭建可评测协议和展示系统闭环，不等价于数据集原生异常标注。

### 5.2 页面中的“视频”为什么不是 MP4？

页面使用 Canvas 逐帧绘制原始背景、轨迹、热力和解释证据。这样可以让同一帧控制同时驱动多层可视化，并支持导出当前画面 PNG。它不是简单播放 MP4，而是一个交互式分析视图。

### 5.3 为什么 Registration 没有原始背景？

Registration 展示的是点云或几何配准诊断，不是 VT-Tiny-MOT 原始视频序列，因此没有原始 RGB 背景。该模块用于证明系统具备配准误差分析和跨模态对齐诊断能力。

### 5.4 学习类方法是否已经接入？

系统展示层已经能接入学习类方法和非学习基线，但当前部分算法仍是占位或轻量实现。方法是否学习、是否为主方法、是否为基线，由 `method_registry.json` 和实验结果表共同说明。后续替换更强算法时，只要输出同样格式的 score JSONL / summary CSV，就能进入同一个 dashboard。

### 5.5 系统与论文流程是否完全一致？

当前系统已经覆盖数据接入、异常协议、个体异常、群体异常、配准诊断、结果汇总、动态可视化、解释证据和报告包交付。但算法部分仍可继续增强，尤其是真实学习模型、真实点云配准输出、更多消融实验和真实异常标注协议。答辩时应把当前系统表述为“论文系统的一版可展示闭环实现”，不要夸大为所有算法都已达到最终论文理想版本。
