# MTF-BA 运行操作文档

## 1. 项目简介

这个项目是一套基于 **VT-Tiny-MOT** 数据集的多模态轨迹异常检测流程，核心思路是：

1. 从 VT-Tiny-MOT 标注中提取 RGB / Thermal 双模态目标轨迹。
2. 将轨迹整理成单目标样本。
3. 为每个样本导出 6 组单模态特征：
   - `route_rgb`
   - `speed_rgb`
   - `shape_rgb`
   - `route_thermal`
   - `speed_thermal`
   - `shape_thermal`
4. 分割训练集 / 验证集。
5. 训练 6 个 LSTM Autoencoder 检测器。
6. 对各检测器打分。
7. 对 6 路分数做集成，输出最终异常分数。

项目当前主要通过根目录下的 Python 脚本运行，不是通过 Web 服务或 GUI 启动。

## 2. 运行前需要准备什么

### 2.1 必备软件

建议准备以下环境：

- Windows 10/11 或 Linux
- Python 3.10 或 Python 3.11
- `pip`
- 建议使用虚拟环境 `venv` 或 Conda
- 如果要用 GPU 训练，需要正确安装 CUDA 版 PyTorch

### 2.2 Python 依赖

这个仓库里目前 **没有完整的 `requirements.txt`**，需要根据代码导入手动安装依赖。按当前代码至少需要：

- `numpy`
- `pandas`
- `torch`
- `scikit-learn`
- `scipy`
- `tqdm`
- `kneed`
- `matplotlib`

建议先在项目根目录创建虚拟环境并安装：

```powershell
cd root\MTF-BA
python -m venv .venv
.venv\Scripts\activate
python -m pip install --upgrade pip
pip install numpy pandas scikit-learn scipy tqdm kneed matplotlib
```

PyTorch 请根据你的机器情况单独安装：

### 2.3 安装 PyTorch

如果你只想用 CPU：

```powershell
pip install torch
```

如果你要用 GPU，建议去 PyTorch 官网根据显卡和 CUDA 版本生成安装命令：

- 官网：[PyTorch Start Locally](https://pytorch.org/get-started/locally/)

安装完成后可测试：

```powershell
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available())"
```

如果输出 `True`，说明当前环境可以用 GPU。

## 3. 数据集目录要求

项目默认会读取下面这个目录：

```text
dataset/VT-Tiny-MOT
```

目录中至少要有：

```text
dataset/VT-Tiny-MOT/
├─ annotations/
│  ├─ instances_00_train2017.json
│  ├─ instances_01_train2017.json
│  ├─ instances_00_test2017.json
│  └─ instances_01_test2017.json
├─ train2017/
└─ test2017/
```

说明：

- `00` 表示 RGB 模态
- `01` 表示 Thermal 模态
- 脚本默认从 `dataset/VT-Tiny-MOT/annotations` 中读取标注
- 如果目录不一致，需要在运行命令里手动传 `--data-root`

当前仓库里已经存在该数据目录，因此默认路径通常可以直接使用。

## 4. 项目主要脚本说明

根目录下常用脚本如下：

- `extract_vt_tiny_mot_trajectories.py`
  - 从 VT-Tiny-MOT 标注提取双模态轨迹
- `export_vt_tiny_mot_individual_trajectories.py`
  - 把轨迹整理成单目标样本 JSONL
- `export_single_modality_features.py`
  - 导出 6 组单模态特征
- `split_train_val_by_sequence.py`
  - 按序列划分 train / val
- `train_single_modality_detector.py`
  - 训练单个检测器
- `train_all_single_modality_detectors.py`
  - 一次训练 6 个检测器
- `score_single_modality_detector.py`
  - 对单个检测器打分
- `score_all_single_modality_detectors.py`
  - 一次对 6 个检测器打分
- `run_single_modality_ensemble.py`
  - 对 6 路检测器输出做集成
- `analyze_single_modality_results.py`
  - 分析最终结果

## 5. 推荐的完整运行流程

下面是推荐的“从零跑通”流程。

### 5.1 第一步：提取 train 轨迹

```powershell
python extract_vt_tiny_mot_trajectories.py --split train
```

默认输出目录：

```text
outputs/vt_tiny_mot_trajectories/
```

会生成：

- `observations_train.csv`
- `trajectories_train.jsonl`
- `summary_train.json`

如果还要提取 test：

```powershell
python extract_vt_tiny_mot_trajectories.py --split test
```

### 5.2 第二步：导出单目标轨迹

先导出 train：

```powershell
python export_vt_tiny_mot_individual_trajectories.py --split train --csv-path outputs/vt_tiny_mot_trajectories/observations_train.csv
```

再导出 test：

```powershell
python export_vt_tiny_mot_individual_trajectories.py --split test --csv-path outputs/vt_tiny_mot_trajectories/observations_test.csv
```

默认输出目录：

```text
outputs/vt_tiny_mot_individual/
```

会生成：

- `individual_trajectories_train.jsonl`
- `individual_trajectories_test.jsonl`
- 对应的 summary JSON

### 5.3 第三步：导出单模态特征

导出 train 特征：

```powershell
python export_single_modality_features.py --jsonl-path outputs/vt_tiny_mot_individual/individual_trajectories_train.jsonl --split train
```

导出 test 特征：

```powershell
python export_single_modality_features.py --jsonl-path outputs/vt_tiny_mot_individual/individual_trajectories_test.jsonl --split test
```

默认输出目录：

```text
outputs/vt_tiny_mot_features/
```

会生成 6 个特征文件，例如：

- `route_rgb_train.pkl`
- `speed_rgb_train.pkl`
- `shape_rgb_train.pkl`
- `route_thermal_train.pkl`
- `speed_thermal_train.pkl`
- `shape_thermal_train.pkl`

test 同理。

### 5.4 第四步：划分 train / val

```powershell
python split_train_val_by_sequence.py --split-features
```

这一步会：

- 从 `outputs/vt_tiny_mot_individual/individual_trajectories_train.jsonl` 按 sequence 划分 train / val
- 同时把 `outputs/vt_tiny_mot_features/` 里的训练特征切成 train / val

输出目录：

- `outputs/vt_tiny_mot_individual_split/`
- `outputs/vt_tiny_mot_features_split/`

### 5.5 第五步：训练 6 个检测器

```powershell
python train_all_single_modality_detectors.py
```

默认模型输出目录：

```text
outputs/vt_tiny_mot_models/
```

每个特征会对应一个子目录，例如：

- `outputs/vt_tiny_mot_models/route_rgb/`
- `outputs/vt_tiny_mot_models/speed_rgb/`

里面通常会有：

- `best_model.pth`
- `normalization_stats.json`
- `train_summary.json`

如果你想指定 GPU：

```powershell
python train_all_single_modality_detectors.py --cuda-device cuda:0
```

如果你机器没有 GPU，也可以直接跑，代码会自动退回 CPU，只是速度会慢很多。

### 5.6 第六步：对检测器打分

先对 test 打分：

```powershell
python score_all_single_modality_detectors.py --split test
```

如果还需要对 train / val 打分，也可以分别执行：

```powershell
python score_all_single_modality_detectors.py --split train
python score_all_single_modality_detectors.py --split val
```

默认输出目录：

```text
outputs/vt_tiny_mot_scores/
```

每个特征每个 split 会有一个目录，例如：

```text
outputs/vt_tiny_mot_scores/route_rgb/test/
```

目录里通常有：

- `embeddings.pkl`
- `reconstruction_loss.pkl`
- `final_scores.pkl`
- `score_records.jsonl`
- `scoring_summary.json`

### 5.7 第七步：做集成

对 test 集成：

```powershell
python run_single_modality_ensemble.py --split test
```

如果已经算了 train / val，也可以继续：

```powershell
python run_single_modality_ensemble.py --split train
python run_single_modality_ensemble.py --split val
```

默认输出目录：

```text
outputs/vt_tiny_mot_ensemble/
```

会生成：

- `aligned_scores_<split>.csv`
- `mean_scores_<split>.csv`
- `max_scores_<split>.csv`
- `complementary_metrics_<split>.csv`
- `ensemble_summary_<split>.json`

### 5.8 第八步：分析结果

以 test 为例：

```powershell
python analyze_single_modality_results.py --split test
```

默认输出目录：

```text
outputs/vt_tiny_mot_analysis/
```

会生成：

- `analysis_full_test.csv`
- `top_50_test.csv`
- `top_50_sequences_test.csv`
- `top_50_categories_test.csv`
- `analysis_summary_test.json`

## 8. 重要参数说明

### 8.1 训练参数

`train_all_single_modality_detectors.py` 常用参数：

- `--hidden-size`
  - LSTM 隐藏层维度，默认 `128`
- `--batch-size`
  - 批大小，默认 `64`
- `--learning-rate`
  - 学习率，默认 `1e-4`
- `--num-epochs`
  - 训练轮数，默认 `100`
- `--early-stopping-patience`
  - 提前停止耐心轮数，默认 `20`
- `--cuda-device`
  - 使用的 GPU，如 `cuda:0`
- `--no-normalize`
  - 不做归一化

示例：

```powershell
python train_all_single_modality_detectors.py --num-epochs 200 --batch-size 32 --cuda-device cuda:0
```

### 8.2 打分参数

`score_all_single_modality_detectors.py` 常用参数：

- `--split`
  - `train` / `val` / `test`
- `--k-neighbors`
  - 近邻数量，默认 `6`
- `--batch-size`
  - 默认 `32`
- `--cuda-device`
  - GPU 设备名

### 8.3 集成参数

`run_single_modality_ensemble.py` 常用参数：

- `--split`
  - `train` / `val` / `test`
- `--top-k`
  - Jaccard 比较时使用的 top-k，默认 `500`

## 9. 运行过程中常见问题

### 9.1 提示找不到数据集

常见原因：

- `dataset/VT-Tiny-MOT` 目录不存在
- 标注文件不在 `annotations/` 下

处理办法：

- 检查目录是否和第 3 节一致
- 或在命令中显式传入：

```powershell
python extract_vt_tiny_mot_trajectories.py --split train --data-root "你的数据集路径"
```

### 9.2 提示缺少某个 Python 包

原因通常是虚拟环境里没有装完整依赖。

处理办法：

```powershell
pip install numpy pandas scikit-learn scipy tqdm kneed matplotlib torch
```

### 9.3 GPU 不可用

检查：

```powershell
python -c "import torch; print(torch.cuda.is_available())"
```

如果返回 `False`：

- 可能安装的是 CPU 版 PyTorch
- 可能 CUDA 驱动或版本不匹配

此时可以先直接用 CPU 跑：

```powershell
python train_all_single_modality_detectors.py
```

代码会自动退回到 CPU。

### 9.4 训练时报 “No training sequences remain after filtering”

原因：

- 过滤后的序列长度不满足最小长度要求
- 尤其是 `shape_*` 特征可能会更短

可以尝试降低最小长度：

```powershell
python train_all_single_modality_detectors.py --min-length 2
```

### 9.5 只跑了 train，没有 test 文件

那后面的 `score --split test`、`ensemble --split test` 就会失败。

需要先补跑：

1. `extract_vt_tiny_mot_trajectories.py --split test`
2. `export_vt_tiny_mot_individual_trajectories.py --split test ...`
3. `export_single_modality_features.py --split test ...`

## 11. 建议的使用方式

如果你是第一次接手这个项目，建议按下面顺序：

1. 先完成环境安装。
2. 先确认 `dataset/VT-Tiny-MOT` 目录存在。
3. 先尝试只运行：
   - `python run_single_modality_ensemble.py --split test`
   - `python analyze_single_modality_results.py --split test`
4. 如果能正常运行，再决定是否从头提取、训练和打分。
5. 如果要完整复现，再按第 5 节的顺序一步一步执行。

这样更稳，也更容易定位问题。

## 12. 一句话总结

这个项目的标准执行顺序是：

**提取轨迹 -> 导出单目标轨迹 -> 导出 6 组特征 -> 划分 train/val -> 训练 6 个检测器 -> 打分 -> 集成 -> 分析。**

如果仓库里的 `outputs` 已经完整，则可以直接从 **打分 / 集成 / 分析** 开始，不一定要从头训练。
